# GlucoTwin AI — Deployment Checklist ✅

Use this before every deployment to avoid common mistakes.

---

## Local Docker (docker-compose up)

- [ ] Docker Desktop is running
- [ ] Copied `.env.example` → `.env` and filled in real values
- [ ] `DB_PASSWORD` is set (not the default placeholder)
- [ ] `JWT_SECRET` is at least 32 characters
- [ ] `REACT_APP_API_BASE` is `http://localhost:8080` (for local)
- [ ] Run: `docker-compose up --build`
- [ ] Visit http://localhost — landing page loads
- [ ] Login with `admin@glucotwin.ai` / `admin123` — dashboard loads
- [ ] Submit a prediction — result page shows risk score and SHAP factors
- [ ] Digital Twin page loads and simulation returns a comparison
- [ ] Chatbot opens and responds (fallback responses if no API key)

---

## Cloud Deployment (Render + Vercel)

### Before pushing to GitHub
- [ ] `.env` is in `.gitignore` (it is — do NOT commit it)
- [ ] `.pkl` model files are committed (`glucotwin-ml/model/*.pkl`)
- [ ] `REACT_APP_API_BASE` in Vercel is set to your actual Render backend URL

### Render (Backend + ML)
- [ ] Both services created from `render.yaml` Blueprint
- [ ] `DB_URL`, `DB_USERNAME`, `DB_PASSWORD` set in Render backend service
- [ ] `CORS_ORIGINS` set to your Vercel frontend URL (no trailing slash)
- [ ] `JWT_SECRET` set (auto-generated by render.yaml — check dashboard)
- [ ] ML service health check passes: `https://your-ml.onrender.com/health`
- [ ] Backend health check passes: `https://your-backend.onrender.com/api/health/ping`

### Vercel (Frontend)
- [ ] `REACT_APP_API_BASE` env var set to `https://your-backend.onrender.com`
- [ ] Deploy successful — no build errors
- [ ] App loads and can log in

---

## Security Before Going Live
- [ ] Changed default admin password from `admin123`
- [ ] `JWT_SECRET` is a random 32+ character string (not the default)
- [ ] `DB_PASSWORD` is strong and unique
- [ ] `ANTHROPIC_API_KEY` stored only in platform secrets, never in code
